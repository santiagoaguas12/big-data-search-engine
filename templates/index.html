¿Cómo funciona el motor?

01

Tokenización

El texto se convierte a minúsculas, se eliminan caracteres no alfanuméricos y se divide en tokens. Se filtran stopwords y se aplica el algoritmo de Porter Stemmer para reducir cada palabra a su raíz.

"Startups innovating" → ["startup", "innov"]
02

Índice Invertido

Por cada término se construye una posting list: diccionario term → {doc_id: tf} donde tf es la frecuencia raw en ese documento. Permite recuperar candidatos en O(1) por término.

"startup" → {1: 3, 4: 1, 7: 2}
03

Scoring BM25

Para cada documento candidato se calcula un score que pondera la frecuencia del término y la longitud del documento. IDF penaliza términos demasiado comunes.

BM25 = Σ IDF(t) · tf · (k1+1) tf + k1(1−b+b·|D|/avgdl)
